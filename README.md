Wordpress Coding Standards Practising
=====================


==============|_|====
What does this project for?
=====================

Applying Wordpress Coding Standards is not hard, we can do it with
just reading documents few times. But how much times you need to
be familar with it so that you can apply it consistently even when
you are working in hard problems? And what will you use to practise
it? Personal projects or customer's projects? This project's
intension is solving that problem.

This project needs:
+ Apache server with PHP extension
+ An account on LeetCode.com


==============|_|====
How to use this project?
=====================

In my personal opinion, to use this project, use should have patient
& a love in programming. If you don't have, I think you should read
documents of Wordpress Coding Standards more times. It is the better
way for you.

To use this project, you do as following:

o Initial Steps:
  + Re-typing PHP scripts in Initial package, then running it to make
sure there are no typo errors.
  + After you can type it fast and exactly without focusing too much in
it, you can modify source code as what you think it is the best.

o Starter Steps:
  + Re-typing PHP scripts in Starter package in LeetCode (following link
in each exercise), then running it to make sure there are no typo errors.
  + After you can type it fast and exactly without focusing too much in it,
you can typing it based on your understanding about algorithm. This will
help you practising applying standards while thinking. Try to optimize the
solution as much as possible.
